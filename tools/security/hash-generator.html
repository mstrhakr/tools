    Hash Generator

    Generate SHA-1, SHA-256, SHA-384, and SHA-512 hashes from text using the browser's built-in WebCrypto API

    How it works

    Hashes are computed using the browser's built-in WebCrypto API (window.crypto.subtle.digest) — no external library, no server call, no data leaves your browser.

    A cryptographic hash function takes any input and produces a fixed-length fingerprint. The same input always produces the same output, but even a single changed character produces a completely different hash. This makes hashing ideal for data integrity verification, password storage, and digital signatures.

    • SHA-256 — 256-bit output. The most widely used; standard for TLS certificates, code signing, and blockchain.
    • SHA-384 / SHA-512 — larger outputs with higher security margin; used in high-assurance environments.
    • SHA-1 — included for legacy/compatibility lookups only. Cryptographically broken — do not use for new security-sensitive work.

    MD5 is intentionally omitted — it has been fully broken since 2004 and should never be used for security purposes. Hashes are one-way: you cannot reverse a hash to recover the original input (pre-image resistance), and finding two inputs with the same hash is computationally infeasible for SHA-2 (collision resistance).
