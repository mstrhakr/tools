    How it works & security notes
    
      
    A JSON Web Token (JWT) is three Base64URL-encoded JSON objects joined by dots: header.payload.signature. This tool splits and decodes all three parts client-side in your browser — nothing is sent anywhere.
    
      
      
        
    • Header — declares the token type (JWT) and signing algorithm (e.g. HS256, RS256).
      • 
        
    • Payload — contains claims: statements about the user or session. Common registered claims: iss (issuer), sub (subject), exp (expiry), iat (issued at), aud (audience).
      • 
        
    • Signature — a cryptographic MAC or digital signature over the header and payload, used to verify the token hasn't been tampered with.
      • 
      
    
      
    Important: this tool decodes the token but does not verify the signature. Any valid or invalid JWT can be decoded. Never trust the claims in a JWT on the client side — signature verification must be done server-side with the correct secret or public key. A decoded token with a forged signature is meaningless without verification.
