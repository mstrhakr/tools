    How it works

    A Unix timestamp is the number of seconds (or milliseconds, in JavaScript) elapsed since January 1, 1970, 00:00:00 UTC. This single integer represents any point in time unambiguously, without timezone or locale complexity — which is why it's ubiquitous in databases, APIs, and log files.

    • Seconds vs. milliseconds: Unix traditionally counts seconds; JavaScript's Date.now() returns milliseconds. This tool auto-detects which you've entered (timestamps > ~2 billion are treated as milliseconds).
    • Y2K38: 32-bit signed integers can only represent timestamps up to 2,147,483,647 — which expires on January 19, 2038 at 03:14:07 UTC. Any system still using 32-bit time_t is at risk. 64-bit systems extend this to the year 292 billion.
    • Negative timestamps represent dates before 1970 and are valid on most systems.
