    Number Base Converter

    Convert between binary, octal, decimal, and hexadecimal — plus any custom base from 2 to 36

    How it works

    Every number system uses the same principle: a digit's value depends on its position (positional notation). In base-N, each position is worth Nposition. Decimal uses 10 symbols (0–9); binary uses 2 (0–1); hex uses 16 (0–9, a–f).

    • Binary (base-2) maps directly onto bits — the native language of hardware. One byte = 8 binary digits.
    • Octal (base-8) was popular in early computing; 3 binary digits = 1 octal digit. Still used for Unix file permissions (chmod 755).
    • Hexadecimal (base-16) packs 4 binary digits into 1 hex digit, making large bit patterns readable. A byte is always 2 hex digits (00–FF). Color codes, memory addresses, and SHA hashes are all written in hex for this reason.
    • Base-36 uses 0–9 and a–z, the maximum expressible with standard alphanumeric characters. Useful for compact IDs.

    Conversion is client-side: input is parsed with JavaScript's parseInt(value, fromBase) and then rendered with Number.prototype.toString(toBase).
