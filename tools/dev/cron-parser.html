    How cron expressions work

    A cron expression is five space-separated fields that define a repeating schedule. From left to right: minute (0–59), hour (0–23), day of month (1–31), month (1–12), day of week (0–7, where 0 and 7 are both Sunday).

    • * — any value
    • */5 — every 5 units (step)
    • 1,3,5 — specific values (list)
    • 1-5 — range

    So */15 9-17 * * 1-5 means “every 15 minutes, between 9am and 5pm, weekdays only.”

    Cron originates from Unix Version 7 (1979), written by Ken Thompson. It's still used virtually everywhere — Linux crontab, Kubernetes CronJobs, GitHub Actions schedules, AWS EventBridge, and more all use this same five-field syntax. Some systems extend it with a sixth seconds field at the front — this tool uses the standard 5-field format.
