    TCP reachability check (ports 443/80) from the backend server. Measures round-trip latency. Requires the self-hosted backend at api.tools.mstrhakr.com.

    How This Works

    Browsers have no raw socket API and cannot send ICMP packets — the mechanism used by the classic ping command. The backend performs a TCP connect probe to the target on port 443 (HTTPS), falling back to port 80. These ports are almost universally reachable, giving better results than ICMP which many firewalls silently block.

    The latency reflects the round-trip from the server, not your browser — useful for checking reachability and baseline latency from a neutral vantage point.

    • < 20ms — same region or data center
    • 20–100ms — cross-continent; typical for most public services
    • > 200ms — distant geography, congested path, or the host is rate-limiting probe connections
