    How This Works

    Reverse DNS maps an IP address back to a hostname using PTR records in the special in-addr.arpa DNS zone. Browsers can't perform DNS queries directly, so the backend resolves the PTR record server-side. The lookup works by reversing the IP and appending the zone: 8.8.8.8 becomes the query 8.8.8.8.in-addr.arpa, which resolves to dns.google. For IPv6, the same principle applies using the ip6.arpa zone with each nibble reversed.

    • PTR records are set by the IP block owner — your ISP must configure reverse DNS for your IP range. You can't set it yourself unless you control the IP allocation.
    • Missing PTR — many residential and dynamic IPs have no PTR record. Some mail servers reject email from IPs without a matching forward-confirmed rDNS (FCrDNS).
    • Multiple PTR records — valid; load balancers and CDN nodes often return several hostnames for the same IP.
