    How This Works

    Browsers enforce CORS (Cross-Origin Resource Sharing), which prevents client-side JavaScript from reading response headers from arbitrary domains. The backend fetches the URL server-side using Go's net/http client and returns the full headers — bypassing the CORS restriction without any privacy concern since the request originates from the server.

    Security headers to look for:

    • Strict-Transport-Security (HSTS) — forces HTTPS and prevents downgrade attacks. Use includeSubDomains with max-age ≥ 31536000 (1 year).
    • Content-Security-Policy (CSP) — restricts which scripts, styles, and resources a page can load, defeating most XSS attacks.
    • X-Frame-Options — prevents clickjacking by blocking iframe embedding. Superseded by CSP frame-ancestors in modern browsers.
    • X-Content-Type-Options: nosniff — stops browsers from guessing content types, preventing MIME-confusion attacks.
    • Permissions-Policy — controls access to browser APIs like camera, microphone, and geolocation on a per-origin basis.
