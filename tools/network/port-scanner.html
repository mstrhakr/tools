    How This Works

    The backend performs a TCP connect scan — it attempts a full TCP handshake on each port. Browsers cannot open raw TCP sockets to arbitrary hosts, so all scanning happens server-side: you're seeing which ports are externally visible from the server's network, not from your local IP.

    • Open — a service responded and completed the TCP handshake. The port is reachable and actively accepting connections.
    • Closed — the host replied with TCP RST. The host is reachable but nothing is listening on that port.
    • Filtered — the probe timed out with no reply. A firewall or security group is silently dropping packets to this port.

    This tool scans a curated set of ~25 well-known ports (FTP, SSH, SMTP, HTTP, HTTPS, RDP, MySQL, etc.). Results reflect external visibility — NAT and load balancers may hide internal service topology. Only scan hosts you own or have permission to test.
