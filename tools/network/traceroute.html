    How This Works

    Traceroute exploits the IP Time-To-Live (TTL) field. Each router decrements TTL by 1; when it reaches zero the router discards the packet and sends back an ICMP Time Exceeded reply — revealing its address and latency. By sending probes with TTL 1, then 2, then 3, etc., each successive hop reveals itself.

    This tool sends TCP probes from the backend server rather than the classic UDP probes used by Unix traceroute. TCP probes traverse NAT and firewalls more reliably. Because the trace runs server-side, you're seeing the network path from the server's perspective, not your ISP.

    • * * * — the router at this hop doesn't respond to probes. The path may still be working fine.
    • Latency spike at one hop but not the next — that router deprioritizes probe responses; actual throughput is unaffected.
    • Most internet destinations are 10–20 hops away. Paths through CDNs or Anycast networks often appear unusually short.
