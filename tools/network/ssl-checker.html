    How This Works

    The backend performs a real TLS handshake using Go's crypto/tls package and inspects the certificate chain returned by the server. Browsers deliberately hide raw TLS details from JavaScript — you can't access the full cert chain or negotiated cipher suite through the standard Fetch API.

    • Subject Alternative Names (SANs) — the hostnames this certificate is valid for. Modern certs use SANs exclusively; the older CN field is ignored by browsers.
    • Certificate chain — a cert is only trusted if it chains up to a root CA in the browser's trust store. Chains have 2–3 certs: end-entity → intermediate → root.
    • Cipher suite — the negotiated algorithm pair for key exchange and encryption. TLS 1.3 mandates AEAD ciphers only; TLS 1.2 allows older options (avoid RC4 and 3DES).
    • Days remaining — Let's Encrypt certs expire every 90 days. Commercial wildcard certs typically last 1 year. Browsers show a hard error for any expired cert.
