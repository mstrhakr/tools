    HTML Entity Encoder / Decoder

    Escape and unescape HTML entities including named, decimal, and hex forms

    How it works

    HTML uses certain characters as markup syntax (<, >, &, "). When you want to display these literally in a page, they must be replaced with HTML entities so the browser doesn't interpret them as code.

    There are three equivalent notations for the same character:

    • Named — &amp;, &lt;, &copy; — human-readable but only available for a defined subset of characters.
    • Decimal — &#38; — works for any Unicode code point.
    • Hex — &#x26; — same as decimal but in base-16; common in XML and security tooling.

    Beyond markup safety, entity encoding is a core XSS (Cross-Site Scripting) prevention technique: if you insert untrusted user input into an HTML page, encoding it ensures any injected <script> tags are rendered as text, not executed as code.
