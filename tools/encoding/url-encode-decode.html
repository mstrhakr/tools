    URL Encode / Decode

    Encode and decode URL components

    How it works

    URLs can only contain a safe subset of ASCII characters. Percent-encoding (defined in RFC 3986) converts any other character to %XX where XX is the byte's hexadecimal value. For example, a space becomes %20 and & becomes %26.

    The three buttons do different things:

    • Encode (component) — uses encodeURIComponent(). Encodes everything except unreserved characters (A–Z a–z 0–9 - _ . ~). Use this for individual query parameter values — it will encode &, =, /, and ?.
    • Encode Full URI — uses encodeURI(). Leaves URI structural characters (: / ? # [ ] @ ! $ & ' ( ) * + , ; =) intact. Use this for a complete URL where you want the structure preserved.
    • Decode — uses decodeURIComponent() to reverse any percent-encoding.

    Common gotcha: HTML forms encode spaces as + (application/x-www-form-urlencoded), but RFC 3986 uses %20. These are different encodings — be aware of which one your system expects.
