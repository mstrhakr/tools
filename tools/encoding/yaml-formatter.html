    How it works

    YAML ("YAML Ain't Markup Language") is a human-friendly data serialization format commonly used for configuration files (Kubernetes manifests, Docker Compose, GitHub Actions, Ansible, etc.). Parsing and conversion use the js-yaml library loaded from CDN — all processing runs in your browser.

    YAML is a strict superset of JSON — valid JSON is valid YAML. Its key feature is that indentation defines structure, not braces or brackets. Common gotchas:

    • Bare booleans: yes, no, on, off, true, false are all parsed as booleans in YAML 1.1 (still used by most parsers). Quote them if you mean the string "yes".
    • Colons in values: key: value: with colon needs quoting — key: "value: with colon".
    • Tabs are forbidden as indentation — use spaces only (2 is conventional).
    • Bare numbers: 0755 is parsed as octal (493 decimal). Quote file-mode strings.
