    How it works

    JSON (JavaScript Object Notation, RFC 8259) is a lightweight text format for structured data. Formatting uses JSON.parse() to validate and parse, then JSON.stringify() with an indent argument to pretty-print. All processing happens in your browser.

    JSON is a strict subset of JavaScript with a few important rules people often forget:

    • Keys must be double-quoted strings — {name: "x"} is invalid JSON; {"name": "x"} is correct.
    • No trailing commas — [1, 2, 3,] is a syntax error.
    • No comments — JSON has no comment syntax; use a superset like JSON5 or JSONC if you need them.
    • Numbers have no integer/float distinction; all are IEEE 754 doubles. Very large integers lose precision — use strings for 64-bit IDs.

    Minifying strips all non-significant whitespace, shrinking transmission size. Syntax highlighting here uses color to distinguish keys, strings, numbers, booleans, and null at a glance.
