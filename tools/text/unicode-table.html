    Unicode & ASCII explained

    ASCII (American Standard Code for Information Interchange, 1963) assigned numbers 0–127 to 128 characters: the Latin alphabet, digits, punctuation, and 33 non-printable control codes. It was the foundation of early computing and is still universally compatible — every UTF-8 file is valid ASCII for the 0–127 range.

    Unicode extends this to over 150,000 characters across ~160 scripts as of Unicode 16.0 (2024), covering every living writing system plus historical scripts, symbols, emoji, and more. Each character has a unique code point written as U+XXXX.

    Encodings:

    • UTF-8 — variable-width: 1 byte for ASCII, 2–4 bytes for everything else. Backward-compatible with ASCII. The dominant encoding on the web (>98% of pages).
    • UTF-16 — 2 or 4 bytes per character. Used internally by JavaScript, Windows, and Java.
    • UTF-32 — always 4 bytes. Simple but space-inefficient; used in some internal processing.
