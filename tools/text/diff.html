    How diff works

    Text comparison uses the Myers diff algorithm (1986), which finds the shortest edit script between two sequences — the minimum number of insertions and deletions needed to transform one text into the other. It runs in O(ND) time, where N is the combined length of both texts and D is the edit distance (number of changes).

    The algorithm operates on lines: it finds longest common subsequences of unchanged lines, then marks everything outside those sequences as added or removed. This is the same algorithm used by git diff, diff, and most code review tools.

    All processing is done entirely in your browser — no text is sent anywhere.
