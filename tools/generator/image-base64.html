    How it works

    Images are read using the browser's FileReader API — nothing is uploaded or transmitted anywhere. The raw binary image data is Base64-encoded in-browser and formatted as a data URI.

    A data URI looks like: data:image/png;base64,iVBORw0KGgo... and can be used anywhere a URL is accepted — HTML <img src>, CSS background-image, SVG embeds, and more.

    Trade-offs to consider:

    • Eliminates one HTTP request — useful for small icons or inline images that are always needed on page load.
    • ~33% larger than the original file — Base64 encodes every 3 bytes as 4 ASCII characters.
    • Not separately cached by the browser — the encoded string is embedded in your HTML or CSS, so it re-downloads with the page every time.
    • Best suited for small, frequently-used assets (favicons, UI icons under ~5 KB). Large images embedded this way hurt page load performance.
